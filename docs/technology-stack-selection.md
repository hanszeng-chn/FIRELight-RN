# 技术选型（建议）

| 层级 | 技术栈 |
|------|--------|
| 跨平台框架 | React Native + Expo |
| 语言 | TypeScript |
| 状态管理 | Zustand |
| 本地存储 | SQLite (expo-sqlite) |
| UI 组件 | React Native Paper / Tamagui |

## 目标平台

| 平台 | 优先级 | 说明 |
|------|--------|------|
| Android | MVP | 首发平台 |
| Web | MVP | 通过 Expo Web 支持 |
| iOS | 后续 | MVP 完成后扩展 |

## 本地存储方案

采用纯客户端架构，数据完全存储在用户设备本地：

- **SQLite**：主数据存储，通过 expo-sqlite 操作，适合移动端和 Web
- **数据导出**：支持导出为 JSON 文件，方便备份和迁移
- **数据导入**：支持从 JSON 备份文件恢复数据

**优势**：
- 零服务端成本，无需后端部署
- 数据隐私性强，完全本地化
- 离线可用，无网络依赖
- 跨平台代码复用

**注意事项**：
- 提醒用户定期导出备份
- 卸载应用会丢失数据
